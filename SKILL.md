# wechat-svg-publisher

Create and publish SVG-style WeChat Official Account drafts and image-message drafts (小绿书). Prefer this skill when the draft body should preserve inline `<svg>` markup, when a Markdown article needs to be rendered with the bundled renderer, or when the post should be a picture-first image list.

## Quick Start

Use the bundled script for the actual publish step:

```bash
node /Users/kyan/.codex/skills/wechat-svg-publisher/scripts/publish-svg-draft.mjs \
  --file /path/to/article.svg \
  --title "文章标题" \
  --cover /path/to/cover.jpg
```

For Markdown input, the script uses its built-in Markdown renderer, then wraps the rendered HTML in an inline SVG by default:

```bash
node /Users/kyan/.codex/skills/wechat-svg-publisher/scripts/publish-svg-draft.mjs \
  --file /path/to/article.md \
  --title "文章标题" \
  --cover /path/to/cover.jpg \
  --theme lapis \
  --highlight solarized-light
```

Use `--dry-run --out /tmp/wechat-svg-preview.html` before a real publish when the SVG was generated by an agent or contains local assets.

For image messages / 小绿书, use Markdown frontmatter `type: image` and place images in the body:

```md
---
title: 图片消息标题
type: image
---

![](./1.jpeg)
![](./2.jpeg)
![](./3.jpeg)
```

Or provide the list explicitly:

```bash
node /Users/kyan/.codex/skills/wechat-svg-publisher/scripts/publish-svg-draft.mjs \
  --file article.md \
  --title "图片消息标题" \
  --type image \
  --image-list "./1.jpeg,./2.jpeg,./3.jpeg"
```

## Inputs

- `.svg`: publish the SVG inline in the draft body. The script strips XML declarations, doctype, `<script>` blocks, and inline event handler attributes before publish.
- `.html`: publish HTML as-is; inline SVG inside the HTML is preserved.
- `.md`: read frontmatter, render Markdown with the bundled renderer, and wrap the rendered article in `<svg><foreignObject>...`.
- Image messages: set `type: image` or provide `image_list`; the script uploads up to 20 local/remote images and creates a WeChat `newspic` draft with `image_info.image_list`.

For SVG input, the script expands simple `.class { ... }` rules from `<style>` into element attributes before publishing because WeChat strips SVG `<style>` blocks in draft previews.

Markdown frontmatter can provide `title`, `cover`, `author`, `digest`, `source_url`, `need_open_comment`, `only_fans_can_comment`, `type`, and `image_list`. CLI flags override frontmatter.

## Required Fields

WeChat draft creation requires a cover material `thumb_media_id`. For SVG-only bodies, pass `--cover` with a JPG, PNG, GIF, or other image file that WeChat accepts. If the body contains uploaded `<img>` or SVG `<image>` assets and no cover is supplied, the first uploaded image is used as the cover.

Image messages require at least one image and support at most 20 images. If no cover is supplied, the first image is used as the cover.

Credentials are resolved in this order:

1. `--app-id` plus `--app-secret`
2. `WECHAT_APP_ID` and `WECHAT_APP_SECRET`
3. export lines in `--tools-md` or `$HOME/.openclaw/workspace/TOOLS.md`
4. skill credential storage at `$XDG_CONFIG_HOME/wechat-svg-publisher/credential.json` or `~/.config/wechat-svg-publisher/credential.json`
5. legacy wenyan-compatible credential storage at `$XDG_CONFIG_HOME/wenyan-md/credential.json` or `~/.config/wenyan-md/credential.json`

The script stores token and upload caches under the skill config directory. It can read legacy wenyan-compatible credentials only as a fallback, but it does not require or call `wenyan-cli`.

## Common Commands

Publish an existing SVG:

```bash
node /Users/kyan/.codex/skills/wechat-svg-publisher/scripts/publish-svg-draft.mjs \
  --file article.svg \
  --title "SVG 排版文章" \
  --cover cover.jpg \
  --author "作者名"
```

Render Markdown with the built-in renderer but publish normal HTML instead of an SVG wrapper:

```bash
node /Users/kyan/.codex/skills/wechat-svg-publisher/scripts/publish-svg-draft.mjs \
  --file article.md \
  --title "文章标题" \
  --cover cover.jpg \
  --no-svg-wrap
```

Set SVG dimensions for Markdown wrapping:

```bash
node /Users/kyan/.codex/skills/wechat-svg-publisher/scripts/publish-svg-draft.mjs \
  --file article.md \
  --title "文章标题" \
  --cover cover.jpg \
  --width 677 \
  --height 2400
```

## SVG Constraints

- Use inline SVG for SVG artwork. Do not rely on JavaScript; WeChat removes or blocks script behavior.
- Prefer CSS animation, SVG `<animate>`, and vector shapes over external assets.
- Keep final content compact. Large base64 images inside SVG can exceed WeChat content limits; use local image files so the script can upload them and replace references.
- Avoid using SVG files as the cover image. Provide a normal image cover, usually JPG or PNG.

Read `references/svg-wechat-notes.md` for design and troubleshooting notes before debugging SVG rendering or publish failures.
